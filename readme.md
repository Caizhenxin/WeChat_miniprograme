# 🌙 微光日记（Glimmer Journal）

> 用生活记录作为入口，用心理流派作为理解框架，用5分钟微行动作为出口。

---

## 🧠 项目简介

「微光日记」是一个基于多心理学流派的轻量化情绪记录与干预工具，旨在帮助用户以低门槛方式理解自身情绪，并通过微小行动实现持续改善。

本项目的核心目标不是替代心理咨询，而是提供一个介于“自我消化”和“正式咨询”之间的**过渡空间**。

---

## 🎯 核心价值主张

用户不需要：

- 学习心理学术语

- 直接进入昂贵的心理咨询

- 承担“我是不是有病”的心理负担

用户可以：

- 写下一段真实感受

- 从多个“心理视角”理解自己

- 做一个仅需5分钟的微行动

- 在不被打扰的情况下逐步变好

---

## 🧩 产品核心模型

系统由五个核心模块构成：

### 1️⃣ 输入层（Input Layer）

用户输入一段自由文本（情绪/事件记录）

---

### 2️⃣ 多流派响应引擎（Interpretation Layer）⭐核心创新

系统基于不同心理学流派生成多种解释视角：

- CBT（认知行为疗法） → 思维纠偏

- ACT（接纳承诺疗法） → 情绪容纳

- 积极心理学 → 优势挖掘

- 叙事疗法 → 问题外化

👉 本质实现：

> 多 Prompt / 多 Agent 并行生成不同风格回应

---

### 3️⃣ 用户选择层（Selection Layer）

用户从多个回应中选择“最有共鸣的一种”

👉 强调：

- 用户参与决策（Human-in-the-loop）

- 不强推单一答案

---

### 4️⃣ 微行动生成（Action Layer）

每个流派回应附带一个：

> ⏱ 5分钟内可完成的具体行为任务

示例：

```json

{

  "task_name": "三栏笔记法",

  "steps": [

    "写下事件",

    "写自动想法",

    "写替代性想法"

  ],

  "duration": 5

}

```

### 5️⃣ 追踪与反馈（Tracking Layer）

- 次日状态回访

- 28天打卡机制

- 行为习惯塑造

---

## 🔁 用户核心流程

写下情绪 → 获得多流派回应 → 选择一种 → 完成微行动 → 次日反馈 → 长期改善

---

## 👤 目标用户

典型用户特征：

- 处于“亚临床情绪困扰”状态（焦虑/压力/低落）

- 不愿或无法进入正式心理咨询

- 有表达欲望，但缺乏结构化理解工具

- 对价格、隐私、标签高度敏感

核心心理需求：

“我想被理解，但不想被定义”

---

## 🚧 MVP（最小可行版本）

### 🎯 第一阶段目标（必须优先实现）

仅实现以下功能：

- 输入一段文字

- 返回3种心理流派回应

- 每种回应附带一个微行动

### ❌ 暂不实现：

- 打卡系统

- 商业化

- 企业微信推送

- 专家系统

- 勋章体系

- 复杂隐私机制

---

## 🏗 技术架构

### 前端（微信小程序）

- 文本输入界面

- 卡片式流派展示

- 简单交互（选择/展开）

### 后端（API服务）

推荐技术栈：

Node.js / Python

REST API

核心接口：

POST /analyze

input:
```json
{
  "text": "今天被领导批评了，很难受"
}

output:

[

  {

    "approach": "CBT",

    "response": "...",

    "action": {

      "name": "...",

      "steps": [],

      "time": 5

    }

  }

]
```
### AI层（核心逻辑）

实现方式：

#### 方案A（推荐）

多次调用大模型：

Prompt 1 → CBT

Prompt 2 → ACT

Prompt 3 → 积极心理

Prompt 4 → 叙事疗法

#### 方案B（进阶）

单次调用 + 结构化输出

---

## 🧠 Prompt设计原则（关键）

每个流派必须具备：

- 明确语气风格（理性 / 温柔 / 观察 / 鼓励）

- 固定结构（回应 + 引导 + 行动）

- 避免说教与评判

---

## 🔐 隐私设计（未来阶段）

- 本地优先存储

- 匿名身份机制

- 云端加密备份（可选）

- 最小数据采集原则

---

## 💰 商业化路径（后期）

在用户“感觉变好之后”再发生：

- 流派解锁（订阅/单次）

- 真人专家批注（9.9元/条）

- 微行动周边（实体产品）

- 高阶咨询引导（非强制）

---

## 🧭 产品定位

不是：

❌ 心理咨询平台

❌ AI聊天机器人

❌ 情绪日记工具

而是：

✅ 一个“多心理模型驱动的行为干预系统”

---

## ⚠️ 关键设计原则

- 降低开口门槛（比写日记更有反馈）

- 消除专业壁垒（不用懂流派）

- 提供即时价值（不付费也有收获）

- 不制造压力（允许退出、不强绑定）

- 强调“陪伴感”而非“治疗感”

---

## 🚀 未来扩展方向

- AI长期记忆（个体成长轨迹）

- 个性化流派推荐

- 社区（匿名经验分享）

- 多模态输入（语音/图片）

- 语音陪伴模式

---

## 🧪 当前阶段结论

本项目的核心挑战不在于技术实现，而在于：

- 多流派输出质量控制

- 用户体验的“轻感”与“真实感”平衡

- 从一次使用到长期留存的转化

---

## 📌 一句话总结

微光日记 = 情绪输入 × 多视角理解 × 微行动输出 × 轻量陪伴系统

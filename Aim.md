
# Aim.md - 微光咨询日记 技术实现指南

本文档为AI编程助手提供清晰的技术上下文，帮助理解项目核心逻辑、数据流、UI美学及关键实现路径。

---

## 一、项目核心逻辑（技术视角）

**一句话定义**  
一个**本地优先、多流派心理回应引擎**，以“情绪日记→流派滤镜→微行动建议”为核心闭环，通过渐进式信任设计（AI→轻付费真人点评）实现商业转化。

**核心数据流**  
```
用户输入日记文本 → 流派选择（默认CBT/ACT） → 调用LLM生成对应流派的回应文案 → 生成5分钟微行动任务 → 记录打卡状态 → 次日微信询问（需集成推送/webhook） → 累计行为触发付费点（解锁新流派/真人点评）
```

**关键状态机**  
- 日记实体：`{ id, user_id, content, emotion_tags, created_at, is_local_only }`
- 回应记录：`{ diary_id,流派类型, llm_response, action_suggestion, user_rating }`
- 打卡系统：`{ user_id, date, action_completed, feeling_score }`（类似Forest浇水模式）
- 付费解锁：`{ user_id, unlocked_flags (CBT/ACT/叙事...), expert_comment_credits }`

## 二、技术架构建议

### 2.1 总体原则
- **隐私优先**：默认仅本地存储，云端仅作为选择性备份（端到端加密）
- **低延迟**：LLM响应时间控制在3秒内（使用流式输出或小型专用模型）
- **跨平台**：优先PWA或React Native，降低安装门槛

### 2.2 推荐技术栈
| 层级 | 选择 | 理由 |
|------|------|------|
| 前端 | React Native Expo / Flutter | 一套代码覆盖iOS+Android，支持本地SQLite |
| 本地存储 | WatermelonDB + MMKV | 高性能离线日记，支持加密 |
| 云端（最小化） | Supabase / PocketBase | 自托管可选，仅处理付费&备份 |
| LLM服务 | OpenAI GPT-4o-mini / Claude Haiku | 低成本，支持高并发生成流派回应 |
| 推送通知 | OneSignal / 微信服务号API | 次日关怀提醒 (需用户绑定微信) |
| 支付 | Stripe + 微信支付 | 9.9元/条专家点评，订阅制解锁全部流派 |

### 2.3 关键模块实现

#### 🔮 多流派响应引擎
**输入**：用户日记文本 + 用户选择的流派ID  
**处理**：使用**结构化Prompt模板**，区分流派逻辑  
**输出**：JSON `{ response_text, action_title, action_steps, duration_minutes }`

示例Prompt结构（以CBT为例）：
```
你是一名【CBT治疗师】。用户写下日记：{diary_content}
请按以下格式回复：
1. 识别一个自动化负性思维（引用原文）
2. 用证据检验挑战它
3. 输出不超过150字的回应文案
4. 生成5分钟微行动任务，包含步骤和预计耗时
返回JSON。
```

**流派库管理**：用配置文件定义每个流派的system prompt，支持动态新增。

#### 🌱 打卡浇水系统（Forest风格）
- 每天完成微行动后，用户点击“浇水”，进度条增长
- 连续28天 → 解锁成就徽章 + 推送周边购买链接
- 技术实现：使用`react-native-reanimated`做浇水动画；本地存储连续日期计数器；跨日检测使用设备时区。

#### 🤫 隐私保护机制实现
- **本地优先**：日记写入`expo-file-system`下的加密SQLite（SQLCipher）
- **面具模式**：用户头像、昵称随机生成（可存储为单独profile）
- **应用伪装**：使用`react-native-appearance` + 自定义启动activity，长按图标时检测密码后跳转真实界面（Android需要修改MainActivity）
- **端到端备份**：用户公钥加密日记内容后上传至S3，私钥留存本地永不发送。

## 三、UI/UX 美观实现指南

### 3.1 设计语言
- **基调**：温柔、留白、微光、纸质笔记本质感
- **色彩**：主色`#FDF8F2`（暖白） + 辅助色`#EADBC6`（旧纸黄） + 强调色`#D4B595`（陶土褐）  
  点缀色：午夜蓝`#2B3A42`（用于文字）
- **字体**：系统默认衬线体（如Times）用于日记正文；非衬线用于界面导航（San Francisco / Roboto）

### 3.2 页面关键组件

#### 📝 日记输入页
- 背景：模拟Figma/Procreate的纸张纹理（可叠加低透明度noise PNG）
- 输入框：无边框，光标为暖黄色，placeholder为“写下此刻的情绪...”
- 提交按钮：圆形浮动按钮，图标为“羽毛笔 + 微光线条”，点击后触发呼吸动画

#### 🔍 流派选择区（回应前）
- 设计四个卡片水平滑动，每个卡片上写流派名称 + 一句话透传（如“CBT · 挑战消极自动思维”）
- 默认高亮“推荐”角标（CBT/ACT），其他卡片半透明

#### 💬 回应展示卡
- 毛玻璃背景（backdrop-filter blur），圆角24px
- 引用用户原文处使用浅黄色底色 + 左竖线
- 微行动建议区域：独立区块，右上角显示计时沙漏图标 + “5min”

#### 🌸 打卡/森林视图
- 28宫格，每日一个种子图标，完成当天浇水后成长为小花/树苗
- 使用Lottie动画展示浇水动作
- 连续打卡七日，动画飘落花瓣

#### 🔐 伪装模式入口
- 设置里开启“应用伪装”后，桌面图标变为“计算器”样式
- 真实界面入口：打开计算器 → 输入特定算式（如1+2+3=）→ 震动后跳转

### 3.3 动效原则
- 所有弹窗、键盘升起采用**温和的缓动曲线**（cubic-bezier(0.2, 0.9, 0.4, 1.1)）
- 反馈振动仅在“完成微行动”和“付费成功”时使用轻触发（~10ms）

## 四、关键接口与外部集成

| 服务 | 用途 | 实现细节 |
|------|------|----------|
| 微信开放平台 | 次日询问推送 + 扫码登录 | 使用模板消息，用户需关注公众号；或集成企业微信API |
| OpenAI API | 多流派回应生成 | 设置temperature=0.7，max_tokens=350；建议使用异步队列（Bull）避免阻塞 |
| 第三方打卡可视化 | Forest风格动画 | 可自绘Canvas或使用`react-native-svg`实现进度环 |
| 支付回调 | 解锁专家点评 | 收到webhook后更新用户`expert_quota`；点评由后台真人心智完成，只返回文字 |

## 五、开发阶段划分（最小可行产品MVP）

### Phase 1 – 单人可用
- [ ] 本地日记存储 + 基础UI（纸张风格）
- [ ] 接入LLM，实现CBT/ACT两个流派的回应生成
- [ ] 展示回应和微行动任务卡片
- [ ] 基础打卡开关（标记完成）

### Phase 2 – 闭环与留存
- [ ] 打卡28天视图（浇水动画）
- [ ] 微信推送提醒（集成模板消息）
- [ ] 累计5次打卡后展示周边商店入口

### Phase 3 – 商业化
- [ ] 支付集成（9.9元解锁真人专家批注）
- [ ] 流派的付费解锁（叙事疗法等）
- [ ] 伪装模式（应用图标伪装）

## 六、对AI编程助手的重要约束

1. **严禁上传未加密的日记数据到任何第三方服务**（除明确允许的用户主动备份）
2. **LLM输出中不得包含“建议就医”等医疗免责声明**（项目面向亚健康人群，需在首次打开时统一展示知情同意书）
3. **所有动画不得使用GIF**，必须用矢量或Lottie保证性能
4. **支持RTL语言**时，纸张纹理需要镜像翻转
5. **如果用户连续3天未打开应用**，只在本地存储标记，不得主动推送“你还好吗”类过度关怀（保护边界感）

## 七、测试重点

- **流派一致性**：同一段日记输入不同流派，输出应具有鲜明的方法论差异（测试用例：被领导批评场景）
- **隐私沙盒**：彻底断开网络后可否正常打开日记并提问（本地LLM可用？初始设计依赖云端API，需提示“联网以获取回应的变通方案”）
- **支付失败回滚**：用户扣款后未收到专家点评时，提供客服入口并自动退券
